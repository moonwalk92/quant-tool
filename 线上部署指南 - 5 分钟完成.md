# 线上部署指南 - 5 分钟完成

## 🎯 目标

把量化交易工具部署到线上，让别人可以通过网址访问。

---

## 📋 第 1 步：创建 GitHub 仓库（2 分钟）

### 1.1 访问 GitHub

打开浏览器，访问：https://github.com/new

### 1.2 创建仓库

- **仓库名：** `quant-tool`
- **描述：** 量化交易工具 - 自动双向交易策略
- **公开性：** ✅ Public（公开）
- 点击 **"Create repository"**

### 1.3 上传代码

在终端执行（替换 `YOUR_USERNAME` 为你的 GitHub 用户名）：

```bash
cd /Users/huan/.openclaw/workspace/quant-tool

# 关联远程仓库
git remote add origin https://github.com/YOUR_USERNAME/quant-tool.git

# 推送代码
git branch -M main
git push -u origin main
```

**完成！** ✅ 代码已上传到 GitHub

---

## 🚀 第 2 步：部署到 Railway（3 分钟）

### 2.1 访问 Railway

打开浏览器，访问：https://railway.app

### 2.2 登录/注册

- 点击 **"Login"** 或 **"Start a New Project"**
- 选择 **"Login with GitHub"**
- 授权 Railway 访问你的 GitHub

### 2.3 创建项目

1. 点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 选择 `quant-tool` 仓库
4. 点击 **"Deploy Now"**

### 2.4 等待部署

Railway 会自动：
- 安装 Node.js 依赖
- 启动服务器
- 分配公网域名

大约需要 2-3 分钟。

### 2.5 获取访问地址

部署完成后：

1. 点击项目卡片
2. 点击 **"Settings"** 标签
3. 找到 **"Domains"** 部分
4. 复制域名，格式类似：
   ```
   https://quant-tool-production-xxxx.up.railway.app
   ```

**完成！** ✅ 服务已部署到线上

## 🛠️ 故障排除

### 问题 1：部署失败

**可能原因：**
- 代码上传不完整
- 依赖安装失败

**解决：**
1. 检查 GitHub 仓库是否完整
2. 查看 Railway 部署日志
3. 重新部署

### 问题 2：网页打不开

**可能原因：**
- 部署未完成
- 端口配置错误

**解决：**
1. 等待 2-3 分钟
2. 检查 Railway 项目状态
3. 查看日志

### 问题 3：价格不更新

**可能原因：**
- WebSocket 连接失败
- 服务器未启动

**解决：**
1. 刷新页面
2. 检查浏览器控制台
3. 重启 Railway 项目
