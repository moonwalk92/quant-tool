## 第五阶段：启动流程

### 5.1 启动顺序

**按顺序执行：**

1. **启动 MT4**
   ```
   [ ] 通过 Whisky/CrossOver 启动 MT4
   [ ] 登录交易账户
   [ ] 打开交易品种图表
   [ ] 启用算法交易
   ```

2. **启动 Python 桥接**
   ```bash
   cd /Users/huan/.openclaw/workspace/quant-tool/mt4-bridge
   source venv/bin/activate
   python mt4_bridge.py
   ```
   ```
   [ ] 桥接服务启动成功
   [ ] 监听端口 5555
   ```

3. **启动 Node.js 服务器**
   ```bash
   cd /Users/huan/.openclaw/workspace/quant-tool
   npm start
   ```
   ```
   [ ] 服务器启动成功
   [ ] 监听端口 3000
   ```

4. **访问网页界面**
   ```
   [ ] 打开浏览器 http://localhost:3000
   [ ] 页面加载正常
   [ ] 显示实时价格
   [ ] 显示账户信息
   ```

5. **启动 EA（可选，如果使用 EA 模式）**
   ```
   [ ] 在 MT4 图表上添加 QuantToolEA
   [ ] 配置 EA 参数
   [ ] 点击"启动交易"按钮
   ```

6. **启动网页交易（如果使用网页模式）**
   ```
   [ ] 点击网页上的"启动交易"按钮
   [ ] 观察双向市价单发送
   [ ] 观察挂单设置
   [ ] 确认一切正常
   ```

### 5.2 启动后检查

**启动后 5 分钟内检查：**

- [ ] 价格实时更新（每秒）
- [ ] 持仓显示正确（2 个：做多 + 做空）
- [ ] 挂单显示正确（2 个：±5 点）
- [ ] 止盈止损设置正确

**启动后 1 小时内检查：**

- [ ] 无异常订单

### 6.3 异常情况处理

**如果发生以下情况：**

#### 回撤接近限制（>15%）

- [ ] 检查是否有异常行情
- [ ] 考虑手动停止交易
- [ ] 评估是否继续策略

#### 系统故障

- [ ] 记录故障时间
- [ ] 截图错误信息
- [ ] 重启系统
- [ ] 检查日志

#### 网络中断

- [ ] MT4 会自动管理持仓
- [ ] 恢复后检查订单状态
- [ ] 必要时手动平仓

#### MT4 崩溃

- [ ] 重启 MT4
- [ ] 重新登录账户
- [ ] 检查持仓和挂单
- [ ] 重新启动 EA
